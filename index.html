How chains are connected

SKI
Subject Key Identifier — a hash of the certificate's own public key. Its ID badge.
AKI
Authority Key Identifier — the fingerprint of the key that signed this certificate. A pointer at its parent.
AKI ⇒ SKI
A link made because child.AKI = parent.SKI. The most reliable match — immune to name collisions and CA key rollover, where a renewed CA keeps the same name but a new key.
DN match
Fallback when key identifiers are missing: the child's issuer name equals the parent's subject name. Weaker, since names can repeat.
✓ / ✗
The label says who the parent is claimed to be; the tick means the signature was actually verified against the parent's public key. A self-signed root has no AKI, or an AKI equal to its own SKI.

Client-side only: parsing and verification happen in this page via WebCrypto. Signature checks cover RSA PKCS#1 v1.5 and RSA-PSS (SHA-256/384/512), ECDSA (P-256/P-384/P-521), and Ed25519 (browser permitting); anything else falls back to key-identifier / name matching, marked accordingly. Private keys are parsed only to compute their public key for matching and never leave the page.
